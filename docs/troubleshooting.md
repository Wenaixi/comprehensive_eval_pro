# 🛠️ 故障排查与 FAQ

本文档总结了 CEP 运行过程中的常见问题及解决方案。

---

## 🌐 1. 网络与接口错误

### 400 Bad Request
- **表现**: 扫描任务或提交时报错。
- **原因**: 可能是服务端返回了空的维度 ID (None)。
- **解决**: 系统已内置过滤逻辑。请确保使用的是最新版本的代码。

### 401 Unauthorized
- **表现**: 提示登录失效。
- **原因**: Token 已过期或在其他设备登录。
- **解决**: 系统会自动重新登录。如果频繁发生，请检查 `accounts.txt` 密码是否正确。

---

## 👁️ 2. 视觉大一统 (Vision 3.0) 相关

### 验证码识别失败
- **建议**: 
  - 确保已在 `configs/settings.yaml` 中正确配置 `siliconflow_api_key`。
  - 检查网络是否能访问 AI 服务。
  - 系统会自动回退到本地 `ddddocr`，如果还不行，请使用手动输入模式。

### 图片模糊导致识别率低
- **原因**: Vision 3.0 为了满足 1MB 限制执行了压缩。
- **解决**: 尝试提供原始尺寸较小的图片，或在配置中适当调高压缩质量阈值。

---

## 📄 3. 文档解析异常

### 班会记录读取失败
- **报错**: `ImportError: No module named 'docx'`。
- **解决**: 执行 `pip install python-docx`。系统默认将此类依赖标记为可选，未安装时会回退为通用生成。

---

## ⚖️ 4. 安全与合规建议

- **敏感信息**: 提交 Issue 前，请务必脱敏日志中的学号、姓名、Token 和 API Key。
- **资源清理**: 定期清理 `runtime/` 目录下的旧日志和临时图片。
- **授权核实**: 再次提醒，请仅在获得明确授权的环境中运行本程序。
