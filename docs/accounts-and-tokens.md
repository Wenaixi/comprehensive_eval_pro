# 账户与身份管理 (Accounts & Auth)

本系统支持大规模账户的批量导入与身份令牌（Token）的自动化管理。

## 1. 账户导入 (`accounts.txt`)

文件路径：`data/accounts.txt`
格式：`账号 密码` (使用空格或制表符分隔)

**示例**:
```text
13800138000 password123
13800138001 password456
```

## 2. 身份令牌管理

### 2.1 自动化认证流
1. 系统读取账户信息。
2. 调用登录接口，若触发验证码，则由 `VisionService` 处理。
3. 成功后获取 `Token` 并缓存至 `storage/state.json`。

### 2.2 Token 缓存与复用
- 系统优先从缓存读取 Token。
- 若 Token 失效（401 响应），系统会自动触发重新登录。

## 3. 安全建议

- **凭证加密**: 建议在生产环境中通过环境变量注入敏感凭证。
- **日志脱敏**: 系统日志已内置脱敏机制，但仍需注意避免在公共环境展示。
